ベクトルタイル提供実験のデータ仕様

2014/8/1  国土地理院

 

1.　はじめに

国土地理院研究開発基本計画に基づき、国土地理院が行う測量成果のベクトルタイル提供事業の検討にあたって、道路中心線ベクトルタイルの提供実験（以下、本提供実験という）を行います。

 

・本提供実験は、ベクトルタイル提供における技術的・施策的課題を国土地理院が把握するとともに、外部からの技術的な提案を受け取り、外部との技術的な議論を通じてベクトルタイルの適切な提供方法を研究開発することを目的とするものです。

・本提供実験の期間は、2014年8月1日から本提供実験終了までとなります。

・本提供実験は、予告なく内容変更したり提供停止したりする場合があります。

・本提供実験のベクトルタイルは基本測量成果と位置付けているものではありませんが、基本測量成果としての提供を検討するにあたって、提供を行うものです。

・本提供実験の利用により生じた損失及び損害等について、国土地理院はいかなる責任も負わないものとします。

 

 

2.　ベクトルタイルの仕様

本提供実験によるベクトルタイルは、地理院タイルと同じ方式で配信します。

http://cyberjapandata.gsi.go.jp/xyz/[dataID]{z}/{x}/{y}.[拡張子]

■GeoJSONタイルのデータ仕様

・拡張子は「geojson」

（「The GeoJSON Format Specification」が定める形式。詳細は次のURLを参照のこと。http://geojson.org/）

・文字コードはUTF-8

・1ファイルに1つのFeatureCollection型のオブジェクトがある。

・個々のオブジェクトはそのFeatureCollection型のオブジェクトの中に配列で入っている。

・個々のオブジェクトには、”geometry”（座標値（経緯度座標））と”property”（属性）が設定されている。
 
 

 

 

 

 

 

 

 

 


【道路中心線】（ラインデータ）

道路域の中心付近で道路の線形形状を代表する線。

属性

属性名

備考

レコードID

rID

ラインデータのデータID
ラインがタイル境界で分割されている場合、このIDが同じことで元が1つのラインだったことが分かる。

整備データ登録日

lfSpanFr

この地物を生成した時点。
地物の実体が存在しはじめた時点ではない。
データ形式はyyyy-mm-dd形式とする。

整備データ削除日

lfSpanTo

この地物の実体が現存しないと判断された時点，もしくは，されるべき時点。
データ形式はyyyy-mm-dd形式とする。

暫定フラグ

tmpFlg

値が「1」の場合，暫定的な地物であることを示す。

出典地理情報レベル

orgGILvl

この地物が，出典元の測量成果として取得されたときの地図的表現精度（位置精度及び表現分類）を図面の縮尺に対する概念として表した数値。以下のどれかが記載されている。
・0
・250
・500
・1000
・2500
・5000
・10000
・25000

地物種別コード

ftCode

地物の区分を示すコードで、４桁の数値で表わされ、上位2桁が大区分を、下位2桁が小区分を示す。

行政コード

admCode

この地物が存在する市区町村のコード。

整備完了日

devDate

この地物について整備作業が完了した年月日。
データ形式はyyyy-mm-dd形式とする。

種別

type

道路を地図記号とした表す場合の表現種別。以下のどれかが記載されている。
・通常部
・庭園路
・徒歩道
・石段
・不明

道路分類

rdCtg

道路の種別。以下のどれかが記載されている。
・国道
・都道府県道
・市区町村道等
・高速自動車国道等
・都市高速道路
・その他
・不明

道路状態

state

道路の状態。以下のどれかが記載されている。
・通常部
・橋・高架
・トンネル
・雪覆い
・建設中
・その他
・不明

階層順

lvOrder

道路及び鉄道の立体交差部における、橋梁の階層の相対順位を表す値。0が最も下層にあることを示す。階層順は、0（トンネル）→0（地上）→0（橋梁）→1（地上）→1（橋梁）。道路状態種別が「橋・高架部」でないにもかかわらず階層順が1以上の場合もあるが、その場合、トンネル部＜雪覆い部＜通常部＜橋・高架部の順で階層が決定される。

名称

name

道路の名称。
複数の名称を設定する必要がある場合は，カンマで区切った文字列として保持する。

通称

comName

道路の愛称や通り名。
複数の名称を設定する必要がある場合は，カンマで区切った文字列として保持する。

道路管理主体

admOfcRd

道路を管理する主体の種別。
・国
・都道府県
・市区町村
・高速道路管理団体
・その他
・不明

幅員区分

rnkWidth

道路の幅員の種類。以下のどれかが記載されている。
・3m未満
・3m-5.5m未満
・5.5m-13m未満
・13m-19.5m未満
・19.5m以上
・不明

実幅員

Width

道路の実際の幅員。

区間ID

sectID

道路の区間を識別するための区間ID。

有料区分

tollSect

走行するために料金が徴収されるかどうかを示す。以下のどれかが記載されている。
・無料
・有料

分離帯区分

medSect

分離帯がある道路であることを示す。分離帯の幅が１ｍ単位で設定される。

自動車専用道路

motorway

自動車専用道路であるかどうかを示す。自動車専用道路である場合に1が設定される。

表示限界地図情報レベル

repLtdLvl

表示縮尺がこの数値を分母とする縮尺より大きい場合に地図記号としての表示を推奨する限界縮尺の分母数値が設定される。

路線コード

rtCode

道路の路線コード。
複数の路線コードを設定する必要がある場合は，カンマで区切った文字列として保持する。
